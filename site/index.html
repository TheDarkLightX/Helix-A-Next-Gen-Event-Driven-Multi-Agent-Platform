Reference product

From intake lanes to dossiers

Helix is positioned beyond Huginn-style event wiring. The desk surface is concrete: bounded source intake, provenance-first evidence, explainable watchlists, and case lifecycles that stay deterministic even when neural systems help with triage.

01 / Sources

Bound every intake lane before it speaks.

Register feeds, website diff monitors, JSON APIs, webhooks, digests, and manual imports with explicit cadence, trust, scope, and health state.

  • Trust-scored source registry with replayable collection boundaries
  • Explicit tags, cadence, and ownership for every source definition
  • Fail-closed collection posture when config or provenance is missing
02 / Evidence

Normalize raw signals into usable evidence.

Every artifact becomes an immutable evidence item with provenance, timestamps, tags, entities, and bounded claim proposals instead of free-form drift.

  • Evidence hashes, timestamps, and source references travel with the record
  • Neural extraction can suggest claims, but cannot silently mutate state
  • Contradictions stay visible instead of being overwritten by summaries
03 / Watchlists

Keep detection explainable.

Watchlists match on deterministic keywords, entities, trust floors, and severity bands so the operator always knows why the desk opened work.

  • Stable hit rationale with exact rule, source, and threshold context
  • Cooldowns and severity tiers to prevent noisy operator fatigue
  • No opaque ranking model deciding whether something matters
04 / Cases

Run dossiers through a finite lifecycle.

Cases collect evidence, claims, and notes behind a deterministic lifecycle: monitoring, ready for briefing, escalated, or closed with a replayable trace.

  • Analyst ownership, linked evidence, and explicit transition history
  • Brief-ready status without bypassing case-state invariants
  • Autopilot can propose follow-up steps, never bypass the case kernel
Positioning
Helix is what you run when you want a personal intelligence agency that stays self-hosted, open source, explainable, and deterministic where it matters.

Functional Core, Imperative Shell

Deterministic state machines are the source of truth. Side effects live at explicit boundaries. This keeps runs auditable, replayable, and safe to automate.

Pure kernels

Finite state machines for policy, guards, and on-chain intent. Deterministic for deterministic inputs.

Adapters

Network, storage, webhooks, and LLM calls happen outside the core. Results are fed back as explicit inputs.

Replay

Store inputs and outcomes to replay runs. Non-deterministic proposals become deterministic evidence once recorded.

Autopilot With Guardrails

Some proposal engines are non-deterministic by design. Helix treats them as untrusted planners: they can suggest actions, collection plans, or case updates, but every proposed step is evaluated through a deterministic guard machine with stable denial reasons.

Modes

  • off: deny autonomous actions
  • assist: require confirmed_by_human=true
  • auto: allow bounded actions within guardrails

High-risk defaults

  • On-chain actions are disabled by default
  • Optional dry-run requirement for on-chain actions
  • Optional mandatory human confirmation for on-chain actions
  • Hard upper bounds on policy command batch size
Design rule
When an LLM is involved, fail closed on ambiguity, missing context, invalid input, or unknown state. Autopilot is power under constraints, not a bypass.

Neuro-Symbolic, Not Vibes

Symbolic checks go first. Neural systems add extraction, scoring, and hypothesis proposals second. If the symbolic side cannot justify the move, the fusion path denies by default.

01
Symbolic entailment

Bounded forward chaining over facts, rules, and triples.

02
Neural scoring

Compute confidence (bounded to [0, 1]).

03
Fusion decision

Allow, review, or deny with deterministic rationale and trace.

Operator Workflow

The operator loop is designed for actual desk work, not demo theater: intake, normalize, detect, escalate, and review autopilot proposals without losing control of the evidence trail.

01
Register sources

Bound cadence, trust, tags, and scope before collection starts.

02
Collect and normalize

Turn incoming artifacts into evidence with provenance, entities, and claims.

03
Match watchlists

Deterministic rules decide whether the desk opens or updates a case.

04
Drive the case

Move dossiers through monitoring, briefing, escalation, or closure.

05
Review autopilot proposals

Accept, deny, or simulate LLM-suggested next steps with explicit rationale.
